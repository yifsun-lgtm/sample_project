# Tutorial 1.4: Amazon RDS PostgreSQL の作成

このチュートリアルでは、ProQuip アプリケーション用のデータベースを AWS RDS (Relational Database Service) 上に作成します。

## 前提知識

- Tutorial 1.1 が完了していること（AWS CLI がインストール・設定済み）
- Tutorial 1.2 が完了していること（OpenShift に接続済み）
- データベースの基本概念（テーブル、SQL）がわかること

## RDS とは？

Amazon RDS は、AWS がデータベースサーバーの運用（バックアップ、パッチ適用、監視）を代行してくれるサービスです。

自分でサーバーに PostgreSQL をインストールして管理する代わりに、RDS を使えば「データベースを使うこと」に集中できます。

## この手順でできること

| データベース | 用途 |
|-------------|------|
| `proquip` | ProQuip アプリケーションのメインデータベース |
| `keycloak` | 認証サーバー (Keycloak) のデータベース |

## 所要時間

約 15〜20 分（RDS インスタンスの起動待ち 10〜15 分を含む）

---

## Step 1: OpenShift クラスタの VPC を確認する

RDS は OpenShift クラスタと同じネットワーク (VPC) 内に作成する必要があります。
同じ VPC にいることで、OpenShift のアプリケーションから RDS にアクセスできます。

```bash
# VPC 一覧を表示
aws ec2 describe-vpcs \
  --query 'Vpcs[*].[VpcId,CidrBlock,Tags[?Key==`Name`].Value|[0]]' \
  --output table
```

**出力例**:
```
+------------------------+-----------------+----------------------------------+
|  vpc-0daf5aadc175682f1 |  10.0.0.0/16    |  cluster-sfw5w-nqwwm-vpc         |
|  vpc-0da78011f6fdea029 |  192.168.0.0/16 |  sfw5w.sandbox2580.opentlc.com   |
+------------------------+-----------------+----------------------------------+
```

`cluster-` で始まる名前の VPC が OpenShift のものです。VPC ID をメモしてください。

---

## Step 2: サブネットの確認と作成

RDS はデータベースの障害対策のために、最低 2 つの異なるアベイラビリティゾーン (AZ) にサブネットが必要です。

### 2.1 既存サブネットの確認

```bash
# OpenShift VPC 内のサブネット一覧
VPC_ID="vpc-0daf5aadc175682f1"  # ← Step 1 で確認した VPC ID に置き換え

aws ec2 describe-subnets \
  --filters "Name=vpc-id,Values=${VPC_ID}" \
  --query 'Subnets[*].[SubnetId,CidrBlock,AvailabilityZone,Tags[?Key==`Name`].Value|[0]]' \
  --output table
```

**アベイラビリティゾーン (AZ) とは？**
AWS のデータセンターの「棟」のようなものです。`ap-southeast-1a` と `ap-southeast-1b` は物理的に離れた場所にあり、一方が故障しても他方で動き続けられます。

### 2.2 サブネットが 1 つの AZ しかない場合

RDS 用に別の AZ にサブネットを追加します:

```bash
aws ec2 create-subnet \
  --vpc-id ${VPC_ID} \
  --cidr-block 10.0.128.0/18 \
  --availability-zone ap-southeast-1a \
  --tag-specifications 'ResourceType=subnet,Tags=[{Key=Name,Value=cluster-sfw5w-rds-private-ap-southeast-1a}]' \
  --query 'Subnet.SubnetId' --output text
```

**CIDR ブロックとは？**
ネットワークのアドレス範囲です。`10.0.128.0/18` は `10.0.128.0` 〜 `10.0.191.255` の約 16,000 個の IP アドレスを使えます。既存のサブネットと重複しない範囲を選びます。

## Step 3: DB サブネットグループの作成

RDS に「どのサブネットを使うか」を教えるための設定です。

```bash
# 2 つのサブネット ID を指定
PRIVATE_SUBNET_1="subnet-03048002ec09e4d10"  # ← 既存の private サブネット
PRIVATE_SUBNET_2="subnet-03f232dd49550bd8b"  # ← Step 2 で作成したサブネット

aws rds create-db-subnet-group \
  --db-subnet-group-name proquip-db-subnet \
  --db-subnet-group-description "ProQuip RDS subnets" \
  --subnet-ids ${PRIVATE_SUBNET_1} ${PRIVATE_SUBNET_2}
```

---

## Step 4: セキュリティグループの作成

「誰がこのデータベースに接続できるか」を制御するファイアウォールルールです。

```bash
# セキュリティグループ作成
SG_ID=$(aws ec2 create-security-group \
  --group-name proquip-rds-sg \
  --description "ProQuip RDS access from OpenShift" \
  --vpc-id ${VPC_ID} \
  --query 'GroupId' --output text)

echo "Security Group ID: ${SG_ID}"

# OpenShift の VPC 内 (10.0.0.0/16) からの PostgreSQL 接続 (ポート 5432) を許可
aws ec2 authorize-security-group-ingress \
  --group-id "${SG_ID}" \
  --protocol tcp \
  --port 5432 \
  --cidr 10.0.0.0/16
```

**ポイント**:
- `--port 5432` は PostgreSQL のデフォルトポート番号です
- `--cidr 10.0.0.0/16` は OpenShift クラスタの VPC 内からのみ接続を許可するという意味です
- インターネットからは接続できません（`--no-publicly-accessible` で作成するため）

---

## Step 5: RDS インスタンスの作成

いよいよデータベースサーバーを作成します。

### 5.1 利用可能な PostgreSQL バージョンの確認

```bash
aws rds describe-db-engine-versions \
  --engine postgres \
  --query 'DBEngineVersions[?starts_with(EngineVersion,`15.`)].EngineVersion' \
  --output text
```

最新の 15.x バージョンを使います（例: `15.18`）。

### 5.2 RDS インスタンスの作成

```bash
aws rds create-db-instance \
  --db-instance-identifier proquip-dev-db \
  --db-instance-class db.t3.medium \
  --engine postgres \
  --engine-version 15.18 \
  --master-username proquip_admin \
  --master-user-password '<ここに強力なパスワードを入力>' \
  --allocated-storage 20 \
  --storage-type gp3 \
  --vpc-security-group-ids "${SG_ID}" \
  --db-subnet-group-name proquip-db-subnet \
  --no-publicly-accessible \
  --backup-retention-period 7 \
  --storage-encrypted \
  --no-deletion-protection
```

**各パラメータの説明**:

| パラメータ | 値 | 意味 |
|-----------|-----|------|
| `--db-instance-identifier` | `proquip-dev-db` | このデータベースの名前（識別子） |
| `--db-instance-class` | `db.t3.medium` | サーバーの大きさ (2 vCPU, 4GB RAM) |
| `--engine` | `postgres` | PostgreSQL を使う |
| `--engine-version` | `15.18` | PostgreSQL のバージョン |
| `--master-username` | `proquip_admin` | 管理者ユーザー名 |
| `--master-user-password` | (秘密) | 管理者パスワード。**必ず安全な場所に記録** |
| `--allocated-storage` | `20` | ストレージ容量 (GB) |
| `--storage-type` | `gp3` | 汎用 SSD（コストと性能のバランスが良い） |
| `--no-publicly-accessible` | — | インターネットからアクセスできないようにする |
| `--backup-retention-period` | `7` | 自動バックアップを 7 日分保持 |
| `--storage-encrypted` | — | データを暗号化して保存 |
| `--no-deletion-protection` | — | Dev 環境なので削除保護はオフ |

### 5.3 作成完了を待つ

RDS インスタンスの起動には **10〜15 分** かかります。

```bash
# ステータスを確認（creating → backing-up → available と変わる）
aws rds describe-db-instances \
  --db-instance-identifier proquip-dev-db \
  --query 'DBInstances[0].DBInstanceStatus' --output text

# 自動で available になるまで待つ（最大 15 分）
aws rds wait db-instance-available \
  --db-instance-identifier proquip-dev-db
echo "RDS is ready!"
```

### 5.4 エンドポイント（接続先アドレス）の取得

```bash
RDS_ENDPOINT=$(aws rds describe-db-instances \
  --db-instance-identifier proquip-dev-db \
  --query 'DBInstances[0].Endpoint.Address' --output text)

echo "RDS Endpoint: ${RDS_ENDPOINT}"
```

**出力例**:
```
RDS Endpoint: proquip-dev-db.cmwtrrewidlo.ap-southeast-1.rds.amazonaws.com
```

このアドレスをメモしてください。アプリケーションの設定で使います。

---

## Step 6: データベースの作成

RDS インスタンスにはデフォルトの `postgres` データベースだけがあります。
ProQuip 用の `proquip` と `keycloak` データベースを作成します。

OpenShift クラスタ内から一時的な PostgreSQL Pod を起動して接続します:

```bash
oc run psql-setup --rm -i --restart=Never \
  --image=postgres:15-alpine \
  -n proquip-dev \
  -- bash -c "
    PGPASSWORD='<RDSパスワード>' psql -h ${RDS_ENDPOINT} -U proquip_admin -d postgres \
      -c 'CREATE DATABASE proquip;' && \
    PGPASSWORD='<RDSパスワード>' psql -h ${RDS_ENDPOINT} -U proquip_admin -d postgres \
      -c 'CREATE DATABASE keycloak;' && \
    PGPASSWORD='<RDSパスワード>' psql -h ${RDS_ENDPOINT} -U proquip_admin -d postgres \
      -c '\l'
  "
```

**成功例**（最後の `\l` コマンドの出力）:
```
   Name    |     Owner     | Encoding |   Collate   |    Ctype    
-----------+---------------+----------+-------------+-------------
 keycloak  | proquip_admin | UTF8     | en_US.UTF-8 | en_US.UTF-8
 proquip   | proquip_admin | UTF8     | en_US.UTF-8 | en_US.UTF-8
 postgres  | proquip_admin | UTF8     | en_US.UTF-8 | en_US.UTF-8
```

`proquip` と `keycloak` が表示されれば成功です。

## トラブルシューティング

### `Cannot find version 15.7 for postgres` エラー

指定したバージョンが利用できません。利用可能なバージョンを確認してください:

```bash
aws rds describe-db-engine-versions \
  --engine postgres \
  --query 'DBEngineVersions[?starts_with(EngineVersion,`15.`)].EngineVersion' \
  --output text
```

### OpenShift Pod から RDS に接続できない

1. セキュリティグループで 5432 ポートが許可されているか確認
2. RDS が OpenShift と同じ VPC にあるか確認
3. RDS が `available` 状態か確認

```bash
# セキュリティグループのルール確認
aws ec2 describe-security-groups --group-ids <sg-id> \
  --query 'SecurityGroups[0].IpPermissions'
```

### RDS を削除したい場合

```bash
aws rds delete-db-instance \
  --db-instance-identifier proquip-dev-db \
  --skip-final-snapshot
```

## 用語集

| 用語 | 説明 |
|------|------|
| RDS | Relational Database Service — AWS のマネージドデータベースサービス |
| VPC | Virtual Private Cloud — AWS 上の仮想ネットワーク |
| サブネット | VPC 内のネットワークの区画 |
| AZ | Availability Zone — AWS データセンターの物理的な区画 |
| セキュリティグループ | AWS のファイアウォール。許可する通信のルールを定義 |
| エンドポイント | RDS への接続先アドレス (ホスト名) |
| gp3 | General Purpose SSD v3 — AWS の汎用 SSD ストレージタイプ |
| CIDR | ネットワークのアドレス範囲の表記法 (例: 10.0.0.0/16) |
