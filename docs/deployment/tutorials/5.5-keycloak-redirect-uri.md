# Tutorial 5.5: Keycloak redirect_uri の修正

このチュートリアルでは、OpenShift にデプロイした ProQuip にブラウザからアクセスした際に表示される「無効なパラメーター: redirect_uri」エラーを修正します。

## このエラーが起きる理由

ProQuip のフロントエンド (Angular) は、ユーザーがアクセスすると自動的に Keycloak のログインページにリダイレクトします。この時、「ログイン後にここに戻してね」という URL (redirect_uri) を Keycloak に送ります。

Keycloak はセキュリティのため、**事前に許可されたURL以外へのリダイレクトを拒否**します。

```
ユーザー → https://proquip-xxx.apps.cluster-xxx.opentlc.com/
         → Keycloak: "ログインして、終わったら↑のURLに戻して"
         → Keycloak: "そのURLは許可リストにないので拒否！"
         → ❌ 無効なパラメーター: redirect_uri
```

開発環境で作成した Keycloak のレルム設定には `http://localhost:4200/*` や `http://localhost:8080/*` しか許可されていないため、OpenShift のルート URL が拒否されます。

## 解決方法

OpenShift のルート URL を Keycloak クライアントの許可リストに追加します。

## 所要時間

約 3 分

---

## Step 1: OpenShift ルート URL を確認

```bash
oc get route proquip -n proquip-dev -o jsonpath='{.spec.host}'
```

出力例:
```
proquip-proquip-dev.apps.cluster-sfw5w.sfw5w.sandbox2580.opentlc.com
```

この URL をメモしてください。

---

## Step 2: Keycloak 管理者トークンを取得して更新

OpenShift クラスタ内から一時的な curl Pod を使って Keycloak の設定を更新します。

```bash
# Route URL を変数に設定（あなたの環境に合わせて変更）
ROUTE_HOST="proquip-proquip-dev.apps.cluster-sfw5w.sfw5w.sandbox2580.opentlc.com"

oc run kc-fix --rm -i --restart=Never \
  --image=curlimages/curl:8.7.1 -n proquip-dev -- sh -c "
ADMIN_TOKEN=\$(curl -s -X POST \
  http://proquip-keycloak:8080/realms/master/protocol/openid-connect/token \
  -d client_id=admin-cli \
  -d grant_type=password \
  -d username=admin \
  -d 'password=<Keycloak管理者パスワード>' | \
  sed -n 's/.*\"access_token\":\"\([^\"]*\)\".*/\1/p')

CLIENT_ID='client-proquip-web'

curl -s -X PUT \
  -H 'Content-Type: application/json' \
  -H \"Authorization: Bearer \${ADMIN_TOKEN}\" \
  \"http://proquip-keycloak:8080/admin/realms/proquip/clients/\${CLIENT_ID}\" \
  -d '{
    \"clientId\": \"proquip-web\",
    \"redirectUris\": [
      \"http://localhost:4200/*\",
      \"http://localhost:8080/*\",
      \"https://${ROUTE_HOST}/*\"
    ],
    \"webOrigins\": [
      \"http://localhost:4200\",
      \"http://127.0.0.1:4200\",
      \"https://${ROUTE_HOST}\"
    ]
  }'

echo ''
echo 'Done. Verify:'
curl -s -H \"Authorization: Bearer \${ADMIN_TOKEN}\" \
  'http://proquip-keycloak:8080/admin/realms/proquip/clients?clientId=proquip-web' | \
  sed 's/,/,\n/g' | grep -E 'redirectUris|webOrigins'
"
```

### パラメータの説明

| パラメータ | 説明 |
|-----------|------|
| `redirectUris` | ログイン後にリダイレクトを許可する URL パターン。`/*` は全パスを許可 |
| `webOrigins` | CORS (クロスオリジン) リクエストを許可するオリジン |
| `CLIENT_ID` | Keycloak 内部の クライアント UUID（`client-proquip-web`） |

### なぜ localhost も残すのか？

ローカル開発 (`npm start`) でも引き続き動作するように、localhost の URL も許可リストに残します。

---

## Step 3: ブラウザで確認

1. ブラウザで `https://<あなたのルートURL>/` にアクセス
2. Keycloak のログイン画面が表示される（エラーなし）
3. `admin` / `admin123` でログイン
4. ProQuip のダッシュボードが表示される

---

## 別の方法: Keycloak Web Console から設定

ブラウザから設定する場合:

1. Keycloak 管理コンソールにアクセス:
   - URL: `https://<ルートURL>/auth/` → admin / <管理者パスワード>
   - または OpenShift 内部: `oc port-forward svc/proquip-keycloak 8180:8080 -n proquip-dev` → `http://localhost:8180`
2. 左メニュー → **Clients** → **proquip-web** をクリック
3. **Valid Redirect URIs** に追加:
   ```
   https://proquip-proquip-dev.apps.cluster-xxx.opentlc.com/*
   ```
4. **Web Origins** に追加:
   ```
   https://proquip-proquip-dev.apps.cluster-xxx.opentlc.com
   ```
5. **Save** をクリック

---

## 恒久対策: レルム設定ファイルの更新

チームメンバーが毎回この手順を踏まなくて済むように、`proquip-realm.json` に OpenShift URL を追加しておくこともできます。

ただし、URL はクラスタごとに異なるため、以下のいずれかを選択してください:

| 方法 | メリット | デメリット |
|------|---------|----------|
| ワイルドカード `*` を追加 | 全 URL を許可、設定不要 | セキュリティリスク（本番非推奨） |
| 環境ごとに手動追加 | セキュリティ確保 | 毎回作業が必要 |
| デプロイスクリプトに組み込み | 自動化 | スクリプトの保守が必要 |

Dev/Staging 環境では、利便性のためワイルドカードを許可するのも選択肢です:

```json
"redirectUris": ["*"],
"webOrigins": ["*"]
```

## トラブルシューティング

### `CLIENT_ID` がわからない場合

Keycloak のクライアント内部 ID を確認:

```bash
oc run kc-id --rm -i --restart=Never --image=curlimages/curl:8.7.1 -n proquip-dev -- sh -c "
ADMIN_TOKEN=\$(curl -s -X POST http://proquip-keycloak:8080/realms/master/protocol/openid-connect/token \
  -d client_id=admin-cli -d grant_type=password -d username=admin -d 'password=<パスワード>' | \
  sed -n 's/.*\"access_token\":\"\([^\"]*\)\".*/\1/p')
curl -s -H \"Authorization: Bearer \${ADMIN_TOKEN}\" \
  'http://proquip-keycloak:8080/admin/realms/proquip/clients?clientId=proquip-web' | \
  sed -n 's/.*\"id\":\"\([^\"]*\)\".*/\1/p'
"
```

### 更新後もエラーが出る

ブラウザのキャッシュ・Cookie を削除してから再試行:
- Chrome: `Ctrl+Shift+Delete` → Cookie とキャッシュを削除
- またはシークレットウィンドウで開く

---

## 用語集

| 用語 | 説明 |
|------|------|
| redirect_uri | ログイン後のリダイレクト先 URL。OIDC/OAuth2 の仕組み |
| CORS | Cross-Origin Resource Sharing — 異なるドメイン間の通信を許可する仕組み |
| webOrigins | CORS で許可するオリジン (ドメイン) のリスト |
| OIDC | OpenID Connect — OAuth2 ベースの認証プロトコル。Keycloak が実装 |
