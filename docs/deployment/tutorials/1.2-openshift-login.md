# Tutorial 1.2: OpenShift クラスタへの接続

このチュートリアルでは、AWS 上の既存 OpenShift クラスタにコマンドラインから接続します。

## 前提知識

- Tutorial 1.1 が完了していること（`oc` コマンドがインストール済み）
- OpenShift クラスタの管理者から以下を受け取っていること:
  - API サーバーの URL
  - 認証トークンまたはユーザー名/パスワード
  - CA 証明書ファイル（自己署名証明書を使っている場合）

## この手順でできること

ローカルの PC から OpenShift クラスタに接続し、アプリケーションのデプロイや管理ができるようになります。

## 所要時間

約 2〜3 分

---

## Step 1: CA 証明書の保存

OpenShift クラスタが自己署名証明書を使っている場合、CA 証明書ファイルが必要です。
管理者から `.crt` ファイルを受け取ったら、以下の場所に保存します。

```bash
# 証明書用のディレクトリを作成
mkdir -p ~/.kube/certs

# 受け取った証明書ファイルをコピー（ファイル名は環境に合わせて変更）
cp /path/to/your-cluster.ca.crt ~/.kube/certs/
```

**証明書ファイルとは？**
HTTPS 通信を安全に行うための「信頼の証」です。ブラウザが Web サイトの正当性を確認するのと同じ仕組みです。OpenShift クラスタが独自に発行した証明書を使っている場合、この CA 証明書を PC に登録することで「このクラスタは信頼できる」と伝えます。

---

## Step 2: oc login でクラスタに接続

### トークン認証の場合（推奨）

管理者から API URL とトークンを受け取っている場合:

```bash
oc login \
  --server=https://api.your-cluster.example.com:6443 \
  --token=eyJhbGciOiJS... \
  --certificate-authority=~/.kube/certs/your-cluster.ca.crt
```

| パラメータ | 説明 | 例 |
|-----------|------|-----|
| `--server` | クラスタの API エンドポイント URL | `https://api.cluster-xxx.sandbox.opentlc.com:6443` |
| `--token` | 認証トークン（長い文字列） | `eyJhbGciOiJS...` |
| `--certificate-authority` | Step 1 で保存した CA 証明書のパス | `~/.kube/certs/ocp-xxx.ca.crt` |

### ユーザー名/パスワード認証の場合

```bash
oc login \
  --server=https://api.your-cluster.example.com:6443 \
  --username=your-user \
  --password=your-password \
  --certificate-authority=~/.kube/certs/your-cluster.ca.crt
```

### 成功例

```
Logged into "https://api.cluster-xxx.sandbox.opentlc.com:6443" as "system:serviceaccount:openshift-config:cluster-admin" using the token provided.

You have access to 75 projects, the list has been suppressed. You can list all projects with 'oc projects'

Using project "default".
Welcome! See 'oc help' to get started.
```

## Step 3: 接続確認

### ノード一覧の確認

```bash
oc get nodes
```

**成功例**（シングルノードクラスタの場合）:
```
NAME                                           STATUS   ROLES                         AGE    VERSION
ip-10-0-9-76.ap-southeast-1.compute.internal   Ready    control-plane,master,worker   4h9m   v1.33.11
```

**成功例**（マルチノードクラスタの場合）:
```
NAME           STATUS   ROLES                  AGE   VERSION
master-0       Ready    control-plane,master   10d   v1.33.11
worker-0       Ready    worker                 10d   v1.33.11
worker-1       Ready    worker                 10d   v1.33.11
```

全てのノードが `Ready` であれば正常です。

### クラスタバージョンの確認

```bash
oc version
```

**成功例**:
```
Client Version: 4.21.16
Kustomize Version: v5.7.1
Server Version: 4.20.23
Kubernetes Version: v1.33.11
```

- **Client Version**: ローカルの `oc` コマンドのバージョン
- **Server Version**: 接続先の OpenShift クラスタのバージョン

### 利用可能なプロジェクト一覧

```bash
oc projects
```

プロジェクト（Kubernetes でいう Namespace）の一覧が表示されます。

---

## Step 4: OpenShift Web Console へのアクセス（オプション）

OpenShift にはブラウザから操作できる Web Console もあります。

```bash
# Web Console の URL を確認
oc whoami --show-console
```

表示された URL をブラウザで開くと、GUI でクラスタを管理できます。

---

## 完了チェック

以下の 2 コマンドが成功すれば完了です:

```bash
# 1. ログイン状態の確認
oc whoami
# → ユーザー名またはサービスアカウント名が表示される

# 2. ノード一覧の確認
oc get nodes
# → 全ノードが Ready 状態
```

## トラブルシューティング

### `x509: certificate signed by unknown authority` エラー

CA 証明書のパスが間違っているか、証明書ファイルが不正です。

```bash
# 証明書ファイルの存在確認
ls -la ~/.kube/certs/

# 証明書の内容確認（BEGIN/END CERTIFICATE があるか）
head -1 ~/.kube/certs/your-cluster.ca.crt
# → "-----BEGIN CERTIFICATE-----" と表示されるべき
```

### `error: You must be logged in to the server (Unauthorized)` エラー

トークンが期限切れまたは無効です。管理者に新しいトークンを発行してもらってください。

### `Unable to connect to the server: dial tcp: lookup ... no such host` エラー

API サーバーの URL が間違っているか、ネットワークに問題があります。

```bash
# DNS 解決の確認
nslookup api.your-cluster.example.com

# ポートの疎通確認
nc -zv api.your-cluster.example.com 6443
```

### ログアウトしたい場合

```bash
oc logout
```

### 複数クラスタの切り替え

複数の OpenShift クラスタを使っている場合:

```bash
# 現在のコンテキスト確認
oc config get-contexts

# コンテキスト切り替え
oc config use-context <context-name>
```

## 用語集

| 用語 | 説明 |
|------|------|
| API サーバー | OpenShift クラスタの管理エンドポイント。全ての操作はここを通る |
| トークン | パスワードの代わりに使う認証文字列。一定期間で期限切れになる |
| CA 証明書 | 認証局 (Certificate Authority) が発行した証明書。通信の暗号化と信頼性の確認に使う |
| ノード | クラスタを構成するサーバー。master (管理) と worker (アプリ実行) がある |
| プロジェクト | OpenShift でのリソースの区画。Kubernetes の Namespace に相当 |
| kubeconfig | `~/.kube/config` に保存される接続設定ファイル。ログイン情報が記録される |
