# Tutorial 2.1: GitHub Secrets の設定

このチュートリアルでは、GitHub Actions の CI/CD パイプラインが OpenShift クラスタや RDS データベースに安全にアクセスできるよう、GitHub Secrets を設定します。

## 前提知識

- GitHub リポジトリの Settings にアクセスできること（リポジトリの Admin 権限）
- Phase 1 が完了していること（OpenShift 接続済み、レジストリ設定済み、RDS 作成済み）

## GitHub Secrets とは？

パスワードや API キーなどの機密情報をソースコードに書かずに、CI/CD パイプラインで使う仕組みです。

- ソースコードには `${{ secrets.MY_SECRET }}` と書く
- 実際の値は GitHub の設定画面で登録する
- ログにもマスクされて表示される（`***`）
- リポジトリの Admin 権限がないと閲覧・変更できない

> **従来との違い**: 従来はデプロイ担当者のPCや共有のExcelファイルにパスワードを保管していた。GitHub Secretsは暗号化保存、ログマスク、権限制御が組み込まれており、パスワードの漏洩リスクを大幅に低減。

## 設定する Secrets 一覧

| Secret 名 | 用途 | 値の取得方法 |
|-----------|------|-------------|
| `OPENSHIFT_SERVER` | OpenShift API サーバーの URL | クラスタ管理者から提供される |
| `OPENSHIFT_TOKEN` | OpenShift への認証トークン | `oc whoami -t` で取得 |
| `OPENSHIFT_CA_CERT` | OpenShift の CA 証明書 (base64) | CA cert ファイルを base64 エンコード |
| `REGISTRY_URL` | OpenShift 内部レジストリの外部 URL | `oc get route` で取得 |
| `RDS_ENDPOINT` | RDS データベースの接続先アドレス | AWS Console or AWS CLI で取得 |
| `RDS_PASSWORD` | RDS の管理者パスワード | RDS 作成時に設定したパスワード |

約 5〜10 分

## 方法 A: GitHub CLI (`gh`) を使う方法（推奨）

### Step 1: GitHub CLI のインストール

```bash
brew install gh
```

### Step 2: GitHub にログイン

```bash
gh auth login
```

以下の選択肢が表示されます:
1. **Where do you use GitHub?** → `GitHub.com`
2. **What is your preferred protocol?** → `SSH`
3. **How would you like to authenticate?** → `Login with a web browser`

ブラウザが開き、表示されたコードを入力してログインします。

確認:

```bash
gh auth status
# → ✓ Logged in to github.com account your-username
```

### Step 3: 各 Secret の値を取得

```bash
# OpenShift トークン
oc whoami -t

# OpenShift CA 証明書 (base64)
base64 < ~/.kube/certs/ocp-sfw5w.ca.crt

# レジストリ URL
oc get route default-route -n openshift-image-registry -o jsonpath='{.spec.host}'

# RDS エンドポイント
aws rds describe-db-instances \
  --db-instance-identifier proquip-dev-db \
  --query 'DBInstances[0].Endpoint.Address' --output text
```

> **従来との違い**: 従来はサーバーのIPアドレスとパスワードを口頭やメールで共有していた。CI/CDでは認証情報を1箇所で集中管理し、パイプラインが自動的に取得する。

### Step 4: Secrets を登録

```bash
REPO="your-username/your-repo"  # ← あなたのリポジトリに置き換え

# OpenShift 接続情報
gh secret set OPENSHIFT_SERVER --repo "${REPO}" \
  --body "https://api.your-cluster.example.com:6443"

gh secret set OPENSHIFT_TOKEN --repo "${REPO}" \
  --body "$(oc whoami -t)"

gh secret set OPENSHIFT_CA_CERT --repo "${REPO}" \
  --body "$(base64 < ~/.kube/certs/your-cluster.ca.crt)"

# レジストリ
gh secret set REGISTRY_URL --repo "${REPO}" \
  --body "$(oc get route default-route -n openshift-image-registry -o jsonpath='{.spec.host}')"

# RDS
gh secret set RDS_ENDPOINT --repo "${REPO}" \
  --body "$(aws rds describe-db-instances --db-instance-identifier proquip-dev-db --query 'DBInstances[0].Endpoint.Address' --output text)"

gh secret set RDS_PASSWORD --repo "${REPO}" \
  --body "your-rds-password"
```

> **従来との違い**: 従来はデプロイスクリプトにパスワードをハードコードしたり、.envファイルをサーバーに手動配置していた。Secretsはコードとは完全に分離され、Gitリポジトリに含まれない。

### Step 5: 登録確認

```bash
gh secret list --repo "${REPO}"
```

**成功例**:
```
OPENSHIFT_CA_CERT   2026-05-30T12:36:09Z
OPENSHIFT_SERVER    2026-05-30T12:36:07Z
OPENSHIFT_TOKEN     2026-05-30T12:36:08Z
RDS_ENDPOINT        2026-05-30T12:36:11Z
RDS_PASSWORD        2026-05-30T12:36:12Z
REGISTRY_URL        2026-05-30T12:36:10Z
```

6 件全て表示されれば完了です。

## 方法 B: GitHub Web UI を使う方法

CLI が使えない場合はブラウザから設定できます。

### Step 1: Settings ページを開く

1. GitHub でリポジトリのページを開く
2. **Settings** タブをクリック
3. 左メニューの **Secrets and variables** → **Actions** をクリック

### Step 2: Secret を追加

1. **New repository secret** ボタンをクリック
2. **Name** に Secret 名を入力（例: `OPENSHIFT_SERVER`）
3. **Secret** に値を貼り付け
4. **Add secret** をクリック

これを 6 回繰り返します。

### 各 Secret の値の取得方法

ターミナルで以下を実行し、出力をコピーして GitHub に貼り付けます:

```bash
# OPENSHIFT_SERVER
echo "https://api.your-cluster.example.com:6443"

# OPENSHIFT_TOKEN
oc whoami -t

# OPENSHIFT_CA_CERT (base64 エンコード必須)
base64 < ~/.kube/certs/your-cluster.ca.crt

# REGISTRY_URL
oc get route default-route -n openshift-image-registry -o jsonpath='{.spec.host}'

# RDS_ENDPOINT
aws rds describe-db-instances \
  --db-instance-identifier proquip-dev-db \
  --query 'DBInstances[0].Endpoint.Address' --output text

# RDS_PASSWORD
echo "your-rds-password"
```

## CI/CD ワークフローでの使い方

GitHub Actions のワークフローファイル (`.github/workflows/*.yml`) 内で、以下のように参照します:

```yaml
steps:
  - name: Login to OpenShift
    run: |
      # CA 証明書を復元
      echo "${{ secrets.OPENSHIFT_CA_CERT }}" | base64 -d > /tmp/ca.crt

      # oc login
      oc login \
        --server=${{ secrets.OPENSHIFT_SERVER }} \
        --token=${{ secrets.OPENSHIFT_TOKEN }} \
        --certificate-authority=/tmp/ca.crt

  - name: Login to Registry
    run: |
      oc whoami -t | docker login \
        ${{ secrets.REGISTRY_URL }} \
        --username unused \
        --password-stdin
```

**ポイント**:
- `${{ secrets.XXX }}` は GitHub Actions が自動的に実際の値に置き換えます
- ログ出力では `***` にマスクされます
- ワークフローファイルはリポジトリに commit して OK（値は入っていないため）

> **従来との違い**: 従来はデプロイスクリプト内に`scp user@server:/path`のように接続情報を直書きしていた。`${{ secrets.XXX }}`形式ならコードレビュー時にも秘密情報が見えず、ログにも表示されない。

## トラブルシューティング

### `gh secret set` で `HTTP 404` エラー

リポジトリ名が間違っているか、リポジトリに Admin 権限がありません。

```bash
# リポジトリ名の確認
git remote get-url origin
```

### Secret の値を更新したい場合

同じコマンドを再実行すれば上書きされます:

```bash
gh secret set OPENSHIFT_TOKEN --repo "${REPO}" --body "新しいトークン"
```

### Secret を削除したい場合

```bash
gh secret delete SECRET_NAME --repo "${REPO}"
```

### OpenShift トークンの有効期限

サービスアカウントトークンは長期間有効ですが、ユーザートークンは期限切れになることがあります。
期限切れの場合は新しいトークンを取得して Secret を更新してください:

```bash
oc login --server=... --token=新しいトークン
gh secret set OPENSHIFT_TOKEN --repo "${REPO}" --body "$(oc whoami -t)"
```

## 用語集

| 用語 | 説明 |
|------|------|
| GitHub Secrets | リポジトリに紐づく暗号化された環境変数。CI/CD でのみ参照可能 |
| base64 | バイナリデータをテキストに変換する方式。証明書ファイルの受け渡しに使う |
| CA 証明書 | Certificate Authority の証明書。通信先の信頼性を検証するために使う |
| `gh` CLI | GitHub の公式コマンドラインツール。リポジトリ管理、PR、Secrets 設定などが可能 |
| マスク | ログ上で `***` に置き換えて秘密情報を隠す GitHub Actions の機能 |
