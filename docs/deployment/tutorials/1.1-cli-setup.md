# Tutorial 1.1: CLI ツールのインストールと設定

このチュートリアルでは、AWS 上の OpenShift (ROSA) にアプリケーションをデプロイするために必要な 3 つのコマンドラインツールをインストールします。

## 前提知識

- ターミナル（コマンドプロンプト）の基本操作ができること
- macOS を使用していること（Linux の場合は手順が異なります）

## この手順で入るもの

| ツール | 何をするためのもの | バージョン (参考) |
|--------|------------------|-----------------|
| **AWS CLI** | AWS のサービス（サーバー、データベースなど）をコマンドで操作する | 2.34.57 |
| **ROSA CLI** | AWS 上の OpenShift クラスタを作成・管理する | 1.2.63 |
| **oc (OpenShift CLI)** | OpenShift クラスタにアプリをデプロイ・管理する | 4.21.16 |

## 所要時間

約 5〜10 分（ネットワーク速度に依存）

---

## Step 1: Homebrew の確認

Homebrew は macOS 用のパッケージマネージャー（ソフトウェアを簡単にインストールするツール）です。

ターミナルを開いて以下を実行します:

```bash
brew --version
```

**成功例**:
```
Homebrew 5.1.12
```

**Homebrew が入っていない場合** は、以下を実行してインストールします:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

インストール後、ターミナルを再起動してから次に進みます。

> **従来との違い**: オンプレミス環境でもパッケージマネージャー（yum, apt）は使っていたが、開発者のローカルPCにクラウド操作用のツールを入れることはなかった。従来はサーバールームに行くか、踏み台サーバーにSSHするだけで作業が始められた。クラウド移行では、ローカルPCがクラウドへの「操作端末」になるため、複数のCLIツールのセットアップが最初のステップになる。

---

## Step 2: AWS CLI のインストール

AWS CLI は、Amazon Web Services をターミナルから操作するためのツールです。

```bash
brew install awscli
```

数分かかる場合があります。完了したら確認します:

```bash
aws --version
```

**成功例**:
```
aws-cli/2.34.57 Python/3.14.5 Darwin/25.5.0 source/arm64
```

> **従来との違い**: 従来はAWSのようなクラウドCLIは不要。サーバールームで直接作業するか、SSHで接続していた。クラウドではAPIで全操作を行うためCLIが必要。サーバーの起動・停止、ネットワーク設定、ストレージ管理など、従来は物理的な作業やGUIコンソールで行っていた操作を、すべてコマンドラインから実行できるようになる。

---

## Step 3: ROSA CLI のインストール

ROSA CLI は、Red Hat OpenShift Service on AWS (ROSA) のクラスタを管理するためのツールです。

```bash
brew install rosa-cli
```

確認:

```bash
rosa version
```

**成功例**:
```
INFO: 1.2.63
```

> **従来との違い**: 従来のオンプレミスではクラスタ管理ツールという概念自体がなかった。サーバーは1台ずつ個別に構築・管理していた。ROSAでは「クラスタ」という単位でサーバー群をまとめて管理し、ROSA CLIでクラスタの作成・スケール変更・アップグレードをコマンド1つで実行できる。

---

## Step 4: OpenShift CLI (oc) のインストール

oc は、OpenShift クラスタ上のアプリケーションを管理するためのツールです。
kubectl (Kubernetes のツール) の拡張版と考えてください。

```bash
brew install openshift-cli
```

確認:

```bash
oc version --client
```

**成功例**:
```
Client Version: 4.21.16
Kustomize Version: v5.7.1
```

> **従来との違い**: 従来はWildFlyの管理コンソール（9990ポート）やSSHでサーバーに直接ログインしてアプリをデプロイしていた。OpenShiftではocコマンドでコンテナの管理・デプロイを行う。WARファイルを手動でコピーする代わりに、`oc apply`や`oc rollout`でコンテナイメージの展開・更新を宣言的に管理する。

---

## Step 5: AWS CLI の初期設定

AWS CLI を使うには、AWS アカウントの認証情報を設定する必要があります。

### 5.1 AWS アクセスキーの取得

1. ブラウザで [AWS Console](https://console.aws.amazon.com/) にログイン
2. 右上のアカウント名 → **Security credentials** をクリック
3. **Access keys** セクション → **Create access key** をクリック
4. **Access key ID** と **Secret access key** をメモ（Secret は一度しか表示されません）

### 5.2 AWS CLI に設定を投入

```bash
aws configure
```

以下の 4 つを聞かれます:

```
AWS Access Key ID [None]: あなたのアクセスキーIDを入力
AWS Secret Access Key [None]: あなたのシークレットキーを入力
Default region name [None]: ap-northeast-1
Default output format [None]: json
```

| 項目 | 入力値 | 説明 |
|------|--------|------|
| Access Key ID | `AKIA...` で始まる文字列 | AWS アカウントの ID |
| Secret Access Key | 長い英数字の文字列 | AWS アカウントのパスワードのようなもの |
| Default region | `ap-northeast-1` | 東京リージョン（最寄りの AWS データセンター） |
| Default output format | `json` | コマンド結果の表示形式 |

### 5.3 設定確認

```bash
aws sts get-caller-identity
```

**成功例**:
```json
{
    "UserId": "AIDAXXXXXXXXXXXXXXXXX",
    "Account": "123456789012",
    "Arn": "arn:aws:iam::123456789012:user/your-user"
}
```

エラーが出る場合、Access Key ID と Secret Access Key が正しいか確認してください。

> **従来との違い**: 従来はサーバーのroot/adminパスワードを担当者が個人管理していた。クラウドではIAMで権限を細かく制御し、アクセスキーで認証する。「誰が」「どのリソースに」「何の操作を」できるかをポリシーで定義するため、退職者のアクセス無効化や権限の最小化が容易になる。また、操作履歴がCloudTrailで自動的に記録されるため、監査対応も改善される。

## Step 6: ROSA CLI のログイン

ROSA CLI を使うには、Red Hat アカウントでログインが必要です。

### 6.1 Red Hat アカウントの作成（持っていない場合）

1. https://console.redhat.com/ にアクセス
2. **Register for a Red Hat account** をクリック
3. 必要事項を入力してアカウントを作成

### 6.2 ROSA CLI トークンの取得とログイン

1. ブラウザで https://console.redhat.com/openshift/token/rosa にアクセス
2. **Load token** をクリック
3. 表示されたトークン文字列をコピー

ターミナルで:

```bash
rosa login --token=ここにコピーしたトークンを貼り付け
```

確認:

```bash
rosa whoami
```

**成功例**:
```
W: OCM API 'https://api.openshift.com' has an outdated trusted URLs configuration
I: AWS ARN:                       arn:aws:iam::123456789012:user/your-user
I: OCM account ID:                xxxxxxxxxxxxxxxxxxxxxx
I: OCM account name:              Your Name
I: OCM account username:          your-username
I: OCM account email:             your-email@example.com
I: OCM organization ID:           xxxxxxxxxxxxxxxxxxxxxx
I: OCM organization name:         Your Organization
```

> **従来との違い**: 従来のオンプレミスではアプリケーションサーバーのベンダー（WildFly、WebLogicなど）に個別にログインする概念はなかった。サーバーにSSHできれば全ての操作が可能だった。クラウドでは「プラットフォーム管理（ROSA/Red Hat）」と「インフラ管理（AWS）」が分離されており、それぞれ別の認証・トークンが必要になる。これにより責任分界点が明確になる反面、管理する認証情報が増える。

---

## 完了チェック

以下の 3 コマンドを全て実行し、エラーが出ないことを確認してください:

```bash
aws sts get-caller-identity
rosa whoami
oc version --client
```

3 つ全てが成功すれば、Task 1.1 は完了です。

## トラブルシューティング

### `command not found` エラー

Homebrew でインストールしたのにコマンドが見つからない場合:

```bash
# Homebrew のパスを確認
echo $PATH | tr ':' '\n' | grep brew
```

`/opt/homebrew/bin` が含まれていなければ、以下を `~/.zshrc` に追加:

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
source ~/.zshrc
```

### AWS `InvalidClientTokenId` エラー

Access Key が無効または期限切れです。AWS Console で新しいキーを作成してください。

### ROSA `token has expired` エラー

トークンを再取得してください:
1. https://console.redhat.com/openshift/token/rosa にアクセス
2. 新しいトークンをコピー
3. `rosa login --token=新しいトークン` を実行

## 用語集

| 用語 | 説明 |
|------|------|
| CLI | Command Line Interface — ターミナルから操作するツール |
| AWS | Amazon Web Services — Amazon のクラウドサービス |
| ROSA | Red Hat OpenShift Service on AWS — AWS 上で動く OpenShift の管理サービス |
| OpenShift | Red Hat が提供する Kubernetes ベースのコンテナプラットフォーム |
| リージョン | AWS のデータセンターの場所（ap-northeast-1 = 東京） |
| IAM | Identity and Access Management — AWS のユーザー・権限管理 |
